Navigation

Design

There are 2 choices on the Map: the AR mode, or the traditional map. Each option allows the user to find the best path from user’s current location, or specified building, to building of their choice. The search bars are located on the top of the map at all times in the map mode, in case the user wants to change their destination, so he/she doesn’t have to go back to the last page. After the user inputs their location, and the next building location, the 3 best routes appear on the screen, as well as the approximated time each route will take the user to arrive at the destination. On the screen, the default highlighted route is the route with the smallest approximated time; however, the user can choose different route if he/she wants to. When the user choose a different path, the application automatically updates the new approximated time of the other routes. After the user decides which path he/she is going to use, the user can click the ‘Preview Route’ button, on the top right underneath the searching location bars. This Preview Route eliminates other two routes, and focus on the chosen route. Furthermore, the searching location bar gets replaced by the navigation information to guide the user to the destination building. The AR mode button on the top right of the preview route screen allows the user to navigate in AR using the phone’s camera. In this mode, an arrow is overlaid on the screen’s view of the world to show the user which direction to move in order to reach their destination. As with all other screens, each feature a ‘back’ button to return the user to the previous screen.
